Tower of Bab-IL

Microsoft, like any other group of one or more persons, has made many "mistakes" (only if you don't learn from them) through out its history. .Net (Framework) was, for sure, not one of them. The dream of a standardized ecosystem (runtime) for all Microsoft (Windows, to start with) things is a good one. The Wikipedia page about the .Net Framewok, tells the story that is not the purpose here. But let's put some definitions in place...

What is .Net?

A software framework that allows creating, with single code base from a myriad of languages, with interoperability between languages. In the end, it consists of a runtime, called Common Language Runtime (CLR, from now on) that excutes a byte code or intermediate language (IL, from now on), called Microsoft Intermediate Language (MSIL, from now on), that is the result of compiling the actual written code. CLR also provides a set of core libraries that catering for common software needs, e.g., File System operations, Networking, etc.

Languages

Although there are .Net compilers for many languages, i.e., compilers that compile a language to MSIL (e.g., Phython for .Net or the defunct Visual J#), Microsoft directly maintains C#, VB.Net and F#. Here, we will focus on these three languages and the native .Net one, i.e., MSIL.
